Secure Software Development


Introduction to the module

The Secure Software Development modules aims to provide a detailed overview of programming languages and risks associated to those, from the perspectives of design and architecture approaches, programming paradigms. It will cover the subject of software testing and it will look into the impact the Operating Systems and Libraries have on software development. In addition to the programming languages, topics such as the distributed systems, APIs, future trends in secure software development, etc. will also be addressed.

In particular, this module will:

  • Introduce the concept of abstraction of programming.
  • Develop an understand of the basic principles of secure software development methodologies.
  • Explore how to perform analysis, program design, software construction and testing required for secure software development.
  • Introduce the basic concepts of software architecture and Software Development Life Cycles such as Agile and TOGAF.
  • Further develop team working skills.
  • Provide an opportunity to reflect on personal development.

Overview of the work completed

Unit overview

The Unit 1 focusses on the review of various software developing methodologies. In this unit we have reviewed the Waterfall and Agile as well as Secure Agile software development approaches as well as looked into the Unified Modelling Language (UML) and how this could support software development. During this week we have also touched upon the subject of various software development standards applicable.

During week 1 of the 'Secure software development' module we have been asked to complete a variety of reading, prepare a UML diagram on the assigned subject as well as to write and sign a team contract for the newly allocated team.

This week's reading material included an overview of the OWASP Top Ten (Mitre, 2017), the introduction into a new software development methodology called Secure Scrum (Pohl & Hof, 2015), ISO 27000 standards (ISO/IEC, 2018), software design patterns (Asghar et al, 2019) and an introduction into software development using Python as well as security considerations the developers must have when using this programming language (Pillai, 2017).

I enjoyed how the authors of the Secure Scrum paper managed to utilise a currently popular software development approach for security related issue development work. The paper not only discussed the theoretical benefits of such methodology, but also practical means of implementing it and provided some great examples to further explain their ideas.

I found this paper most relevant to my current role as the Project Manager/Business Analyst. Depending on the type of project, we may deliver some using the waterfall methodology and others using the Agile methodology. When using the Waterfall, I always make sure that in the project design phase we cover the security aspects. However, when using the Agile method I find that the security aspects are often missed during the many iterations or you start off with having them, but in the subsequent iterations you loose the sight of it. The suggestions made in the paper on using the Security tags to tag to the issues in the iteration which then link to the security vulnerabilities is a great way of not loosing the track of the security exposures and tracking them to completion along many sprints.

The other reading material was interesting too, however, some of it was a little less relevant to my current job, although, still informative.
