The Human Factor


Introduction to the module

It is popular to say that humans are the weakest link in the security and application success. This module aims to develop understanding on security risks and issues associated to the human interactions with computing systems. It will also help develop the skills and knowledge required to manage and mitigate cyber risks in socio-technical systems.

Overview of the work completed

Unit overview

This unit aims to explore the human behaviour and its implications when designing the computing systems with respect to cyber security, discuss the human limitations and capabilities and their impact on usable security, discuss the concept of insider threat.

Over the years, the smartphone popularity has only been growing. These devices now store large quantities of sensitive user data such as location information, contacts, etc. and so it is important to protect it. To be able to secure the data, it is necessary to understand the smartphone user behaviour so that effective user policies and training can be developed.

The security issues in the smartphones can be introduced, among others, by insecure or unfriendly (malicious) applications installed by the user, the loss or theft of the device, connections to insecure WIFIs, etc. There are many tools available to the user to help with these risks: mobile antivirus, backup tools, passwords, etc. However, these tools may not be enough to protect the sensitive data if the actual user behaviour is unknown.

To obtain more information on the smartphone user behaviour, an online survey was created and conducted on the users in China in 2016. Its aim was to find out more about the user attitudes and actions towards the smartphone security and compare it against the demographic user data (user age, sex, occupancy, etc.).

The 338 survey participants included both women and men, those with higher education as well as those working for private and public organisations.

The survey analysed three aspects: the user behaviour related to security on their smartphones, the user behaviour when implementing security add-ons (passwords, antivirus, etc) and the data backup related user actions.

The survey had found that:

  • Less than 20% of users analyse the permissions needed by the application before installing it.
  • More than 40% of users do not download applications from secure sources.
  • Only 21% of users always log out of applications after using it.
  • Over 40% of participants have clicked on links in emails prior to investigating it.
  • Only 46% of users check for smartphone software updates.
  • Most users connect to unsecure wifi occasionally.
  • Backups are never or occasionally used by three quarters of the participants.

The same survey has also shown that users are likely to install security related software on their smartphones such as antivirus, use screen locks and passwords and turn off their location services. However, over 40% of participant passwords were weak.

The information provided by the users was then compared to the demographic user data (sex, age, etc).

  • Male smartphone are more likely to install risky application, however, they are also better of cleaning personal information from old devices and logging out of applications.
  • Students are better at keeping their devices updated than those employed.
  • Employed participants change their mobile devices more often and are more likely to not dispose of personal data correctly.

This study confirms that the smartphone users are lacking security training and provides some useful knowledge into how the security training and policies could be improved by considering the user from different demographics security related behaviour.
