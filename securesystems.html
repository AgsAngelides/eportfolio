Secure Systems Architecture


Introduction to the module

This module aims to introduce the concepts and principles of the Secure Systems Architecture (SSA) and it involves an introduction into basic architecture of modern operating systems, the relationship between it and the Distributed systems, secure system coding practices, the role of virtualisation in SSA.
We will also look into the current and future architecture challenges, limitations and opportunities.

Overview of the work completed

Unit overview

The aim of unit 1 is to introduce us to the processes, threads and schedulles as well as the distributed systems. We are also going to be learning how to set up and use OS containers (e.g. Dockers).

I was unable to install the Docker using the guidance provided by the university. The .io file appears to be an older Docker version so I used docker-ce instead which appeared to work better. The installation guidance provided by the uservisity did not work and got hung at one point.

After a while, debian based VM would not work, so I restarted my work, and installed a Ubuntu on my Virtual box and then used an article by Parveez (2022) to guide me through docker installation and testing on Ubuntu.

I then tried to follow the link to the page provided to learn how to use containers for Python code, however, I found the instructions very unclear. It was not clearly stated how exactly to build the dockerfile nor how to run it. All that prior knowledge was assumed to already exist by the recommended page Author.

After watching some more videos on youtube on how to set up a Docker file, I was able to create a docker file using the university provided link and run it.

When trying to run the docker container, the requirements file as per the link provided by the university appears to also contain an incorrect Flask version which no longer supports 'Escape' module, so I also needed to update the requirements of the docker contaner to use flask 2.1.0 rather than 1.1.1.

This unit's reading included and acrticle on the problems experienced by the distributed systems (Garcia-Valls et al, 2017), a paper on the evolution of distributed systems (Salah et al, 2016), a paper on walware analysys collected over 5 years (Lever et al, 2017) and the chapters 2, 4, 5 and 7 of the Architectural Transformations in Network Services and Distributed Systems book (Luntovskyy & Spillner, 2017).

I have enjoyed finding more about the distributed systems problems, especially since it is becoming a new norm. Also, many of the issues mentioned by the authors are applicable to other systems, such as: sharing bugs in a timely fashion, responding quick, etc.

The paper on the distributed systems evolution was very informative too and it provided a good comparison between SOAs, microservices, client-server based systems, etc. as well as a good overvie of the potential issues faced by the microservices. This paper also expanded well on the unit's lecturecast material.

The paper analysing various mawlare types and their occurences was particularly interesting, since it has provided an insight into the domain blackilisting usefullness, i.e. that it was not likely to cover malicious domains used by the current malware due to the delay of the domain being discovered as malicious and being reported. Additionally, it provided an overview of how wide spread the malware introduced by Potentialy Unwated Programs (PUPs) is on even some very well known domains such as Amazon.co.uk and that potentially, there is a need to have PUP related security policies.
