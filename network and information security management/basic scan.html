Difficulties I had and ways I have overcome it:

  • My windows 10 machine did not have the whois package installed, so I had to install it before I could look up who owns the domain we were given.
  • I was getting confused between Linux and Windows command line statements. Many online resources do not make it clear upon first look which of the two OS it applies to.
  • Nslookup command in command prompt for looking up the MX record of the website, does not work on my machine. Potentially a configuration issue. I had to use an online tool instead to obtain the information.
  • Trying to run a traceroute command directly in the command line would not work. I had to cd first to the folder where the tracert exe was stored to be able to use this command.
